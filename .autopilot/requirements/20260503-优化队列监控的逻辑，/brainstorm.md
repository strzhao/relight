# Brainstorm — 队列监控优化

## Q&A 结果

### Q1: 视觉伴侣
- **选择**: 需要线框图，在设计阶段确认 UI 方向

### Q2: 实时更新方案
- **选择**: SSE (Server-Sent Events)
- **理由**: 实时性好，双向通信非必须，Hono 原生支持 stream()

### Q3: 导航结构
- **选择**: 侧边栏 + 详情布局
- **理由**: 经典管理后台风格，左侧队列列表，右侧详情面板，适合后续扩展

### Q4: 操作能力
- **选择**: 仅作业级别查看详情
- **说明**: 不做 cancel/retry/pause/resume 等变更操作，纯监控展示

### Q5: 整体方案
- **选择**: 方案 A — 轻量 SSE 通道
- **说明**: 每个队列独立 SSE 端点，BullMQ 事件驱动推送，无额外依赖

## 技术决策摘要

| 决策点 | 选择 | 影响 |
|--------|------|------|
| 实时机制 | SSE per queue | Hono `stream()` 推送，前端 `EventSource` |
| 页面布局 | 侧边栏 + 详情 | 左侧 NavLink 队列卡片，右侧动态路由 `[queueName]` |
| 操作范围 | 仅查看详情 | 无 mutation API，纯展示 |
| 队列范围 | 先做 scan + analyze | daily-selection 预留 UI 但标注"即将支持" |
