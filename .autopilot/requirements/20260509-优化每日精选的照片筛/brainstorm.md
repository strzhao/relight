# Brainstorm — 每日精选筛选逻辑优化

## 现状速览（探索结论）

**候选池规则**（`apps/backend/src/jobs/daily-selection.ts:48-62`）
- 严格匹配「历史上的今天」（`strftime('%m-%d', takenAt) = 今日月日`）
- 必须已有 AI 分析记录
- 连拍组只取代表
- 按 `aestheticScore DESC` 截 top 20

**评选目标**（`prompts/v2/daily/select/system.txt`）
- AI 评选标准：真实感 / 情感能量 / 生活趣闻 / 时空厚度 / 人际温度
- AI 失败 → fallback 到最高分照片

**未被使用的信号**
- composition.score / motionScore / transcript 等 AI 已分析的维度
- 拍摄设备（cameraMake 字段没建）
- GPS 位置（字段没建）
- 用户反馈（无点赞/跳过/浏览统计）
- pickDate UNIQUE 但**没有**「N 天内不重复同一张」机制

**结构性风险**
- 历史上的今天 = 月日匹配 → 用户某月某日没拍过照片 → 候选池空 → 当天直接跳过精选（用户感知为「今日没回忆」）
- 美学评分是 AI 给的，会偏向「画面好看」，但「有价值的回忆」不等于「画面好看」
- 单张照片 → 信息密度低，无法呈现「一段时光」

## 优化方向矩阵

### 方向 A：扩展候选池 — 让"今日"不会空

| ID | 方案 | 收益 | 成本 |
|----|------|------|------|
| A1 | 月-日 ± N 天（N=3）模糊匹配 | 池子稳定，不会空 | 弱化"这天"仪式感 |
| A2 | 多策略叠加（历史上的今天 + 周年 + 节日 + 季节回望 + 随机） | 回忆维度丰富 | 需要"模式调度"逻辑 |
| A3 | 候选空时回退（扩窗 / 上周精选 / 完全随机） | 兜底不留白 | 增加状态机复杂度 |
| A4 | 主题驱动池（按"和家人"、"旅行"等主题轮换） | 有节奏感 | 需要主题分类（依赖标签质量） |

### 方向 B：升级排序权重 — 不只看美学评分

| ID | 方案 | 收益 | 成本 |
|----|------|------|------|
| B1 | 多维度组合分（美学 + 情感强度 + 构图 + 时空厚度） | 充分利用已分析信号 | 需要权重调试 |
| B2 | "记忆价值分"（人物存在 / 互动场景 / 罕见事件加分） | 接近"有价值回忆"语义 | 依赖标签精度 |
| B3 | 旧照片加成（年龄越大权重越高） | 真正的"翻旧账" | 新照片可能从来选不上 |
| B4 | 罕见度加分（很少拍这种场景的照片优先） | 多样性提升 | 需要全库统计 |

### 方向 C：避免重复 — 同一回忆不刷屏

| ID | 方案 | 收益 | 成本 |
|----|------|------|------|
| C1 | N 天内不重复同一张 photoId | 简单有效 | 候选少时仍会循环 |
| C2 | 同一连拍组 / 同一事件兄弟也算"已展示" | 进一步去重 | 需要事件聚类 |
| C3 | 控制人物 / 地点出现频率 | 多样性强 | 需要人脸/地点识别 |

### 方向 D：引入用户反馈 — 闭环学习

| ID | 方案 | 收益 | 成本 |
|----|------|------|------|
| D1 | 加「喜欢 / 跳过 / 收藏」按钮 + 影响后续筛选 | 长期学习用户偏好 | 需要前后端字段 + UI |
| D2 | 标记"特殊日子"（生日 / 纪念日）→ 该日强化关联 | 高情感价值 | 需要用户主动配置 |
| D3 | 浏览统计（viewCount / dwellTime）作为隐式信号 | 无需用户主动操作 | 需要前端埋点 |

### 方向 E：单张 → 多张「故事化」

| ID | 方案 | 收益 | 成本 |
|----|------|------|------|
| E1 | 维持单张精选（现状） | 简单清晰 | 信息密度低 |
| E2 | 主图 + 2-3 张配图（同事件 / 同主题） | 沉浸感强 | 需要事件聚类 + UI 改造 |
| E3 | 故事线（多张串成一段叙事） | 最强情感冲击 | 工程量大，AI 写作成本上升 |

### 方向 F：候选输入信号 — 字段补齐

| ID | 方案 | 收益 | 成本 |
|----|------|------|------|
| F1 | 给 photos 表加 GPS / cameraModel | 后续地点/设备维度可用 | DB 迁移 + EXIF 重读 |
| F2 | 把 composition.score / motionScore 物化进候选摘要 | 让阶段 1 AI 更全面 | prompt 调整 |
| F3 | 用人脸聚类做"人物维度"（脱离 AI 标签） | 真正的人物多样性 | 引入人脸库（重量级） |

## Q&A 记录

### Round 1 — 价值取向

- Q: 「有价值的回忆」最重要的是哪个维度？
  - A: **时间跨度 / 久远感**（旧照片更珍贵，新照片权重低）

- Q: 「历史上的今天」候选池为空时怎么处理？
  - A: **多策略叠加**（历史上的今天 + 同月份 + 同季节 + 完全随机老照片）

- Q: 防重复刷屏的代价？
  - A: **最简单 — 单张照片去重**（N 天内同一 photoId 不再次精选）

- Q: 是否引入用户反馈？
  - A: **本期不做**

### Round 2 — 实施细节

- Q: 多策略源的混合方式？
  - A: **平等加权混采** — 四个源都占名额进入候选池，由 AI 评选

- Q: 久远度加权强度？
  - A: **温和加成** — 5 年前 +25% 左右，5 年以上缓慢递增封顶

- Q: 去重窗口？
  - A: **30 天**

## 决策汇总

| 维度 | 决策 |
|------|------|
| 候选池策略 | 4 源（历史上的今天 / 同月份 / 同季节 / 久远随机）平等加权混采 |
| 排序加权 | 美学评分 × 久远度乘数（开根号曲线，封顶 1.6） |
| 去重窗口 | 30 天 |
| 用户反馈 | 本期不实现 |
| AI prompt 调整 | 候选摘要附加 source / yearsAgo 标签 + system 提示多样性约束 |
