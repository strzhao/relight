# 需求探索摘要

## 项目上下文

- **项目**：relight — AI 照片分析管理平台
- **后端**：Hono + Drizzle (SQLite) + BullMQ + qwen3.6-35b AI 视觉模型
- **前端**：Next.js 15 + React 19 + Tailwind v4 + shadcn/ui 组件
- **照片目录**：~/nas-photos（4 个子目录：历史照片、精彩记忆、来自 iPhone 12 Pro Max 的备份、14 年前照片）
- **当前状态**：后端 API 和 AI 管线已就绪，前端仅有占位页面

## Q&A 结果

### Q1: 「AI 打开情况」含义
- **选择**：两者都需要
- 需要同时展示照片维度的 AI 分析状态和 AI 服务运行状态

### Q2: 管理操作
- **选择**：全部四项
  - 触发扫描
  - 查看分析结果
  - 队列/任务监控
  - 服务健康检查

### Q3: 部署位置
- **选择**：现有 Next.js web 应用
- 在 apps/web 中新增管理页面

### Q4: 服务指标
- **选择**：全部四项
  - 队列任务统计
  - 存储源状态
  - AI 调用统计
  - 系统资源

### Q5: 实现方案
- **选择**：方案 C — 渐进式（先简后全）
- 首期交付核心仪表盘 + 照片分析列表，预留扩展点

### Q6: 视觉伴侣
- **选择**：是，启动视觉伴侣
- 服务已启动在 http://localhost:51845

### Q7: 数据刷新方式
- **选择**：手动刷新
- 页面加载时获取数据，用户点击刷新按钮手动更新

### Q8: 后端 API
- **选择**：需要新建后端 API
- 为仪表盘新增必要的统计接口

### Q9: 技术方案
- **选择**：方案 B — 多页面管理后台
- /admin layout + 侧栏导航，每个功能独立页面

## 方案选择

- **选定方案**：多页面管理后台（方案 B）
- **选择理由**：结构清晰、扩展性强、适合后续持续迭代、渐进式演进
- **被排除方案**：
  - 方案 A（Tab 单页）：功能增多后 Tab 过多，单页性能下降
  - 方案 C 的简化版本已融入方案 B 的首期交付范围

## 约束和偏好

- 中文用户界面
- Tailwind v4 + shadcn/ui 组件体系
- 手动刷新（非实时推送）
- 首期交付：仪表盘首页 + 照片分析列表 + 队列监控 + 系统健康
- 需要新建后端 API 提供真实数据
