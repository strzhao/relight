## 修复 3：骨架屏列数与实际列数一致

### 检查步骤

1. 访问 `/photos`，在页面完全渲染前（首次请求 in-flight 时），观察骨架屏展示的列数
2. 等待数据加载完成，观察实际照片网格的列数

### 验收标准

- [ ] **1920px 宽度**：骨架屏列数 = 数据加载后的实际网格列数（视觉无跳动）
- [ ] **1440px 宽度**：骨架屏列数 = 数据加载后的实际网格列数
- [ ] **768px 宽度（平板）**：骨架屏列数 = 数据加载后的实际网格列数
- [ ] **375px 宽度（手机）**：骨架屏列数 = 数据加载后的实际网格列数
- [ ] 骨架屏与真实网格之间**无明显的布局跳变**（CLS 感知为 0）

### 反例（不通过）

- 骨架屏显示 2 列，数据加载后显示 4 列 → **不通过**
- 骨架屏高度与照片卡片高度差异 > 10% → **不通过**

## 修复 4：PhotoCard 缩略图加载失败态

### 检查步骤

1. 打开 Chrome DevTools → Network → 右键任意图片请求 → "Block request URL"
2. 刷新 `/photos` 页面
3. 观察被 block 的图片卡片的显示状态

### 验收标准

- [ ] 缩略图加载失败时，卡片内显示 **ImageOff 图标**（或其他占位图标，可视觉辨认）
- [ ] 图标下方或旁边显示**"加载失败"文字**（中文）
- [ ] 失败态不破坏卡片整体布局（不出现空白大块或溢出）
- [ ] 失败态卡片与正常卡片**保持相同尺寸**（网格不塌陷）

### `<img alt>` 属性检查

1. 打开 DevTools → Elements → 找到任意照片的 `<img>` 标签
2. 检查 `alt` 属性值

- [ ] `<img alt>` 属性值 = 文件的 **basename**（如 `DSC_0001.jpg`），**不是**空字符串
- [ ] alt 值不包含完整绝对路径（不含 `/`），只有文件名部分

## 修复 2（补充手测）：sentinel 区域行为

### 检查步骤

1. 确保数据库有 > 20 张照片（触发分页）
2. 访问 `/photos`，等待首屏加载
3. 使用 DevTools Throttling 将网络设为 "Slow 3G"
4. 缓慢滚动到页面底部

### Idle 状态验收（核心）

- [ ] **首屏加载完成后、滚动到底部之前**：sentinel 区域**不显示**"上滑加载更多"文字
- [ ] **整个使用过程中**：页面任何位置不出现"上滑加载更多"文字

### Loading 状态验收

- [ ] 滚到底部触发加载时，sentinel 区域显示 **spinner**（旋转加载动画）
- [ ] spinner 可见时，**不显示**"加载失败，点击重试"按钮

### 错误状态验收（Slow 3G 下手动制造错误）

1. 打开 Network 面板，在分页请求 in-flight 时，断开网络（DevTools → 勾选 "Offline"）
2. 观察 sentinel 区域

- [ ] 请求失败后，sentinel 区域出现**"加载失败，点击重试"按钮**
- [ ] 按钮**可点击**（可见且不 disabled）
- [ ] 点击后，重新触发加载请求（Network 面板可见新请求）
- [ ] 恢复网络后，点击重试，数据正常追加

## React Hydration 健康度检查

### 检查步骤

1. 访问 `/photos`
2. 打开 DevTools → Console 面板
3. 刷新页面，观察 Console 输出

### 验收标准

- [ ] Console 中**无 React hydration warning**（即无 `Warning: Text content did not match`、`Warning: Prop … did not match` 等）
- [ ] Console 中**无 React 18/19 严格模式警告**
- [ ] Console 中**无 TypeError / ReferenceError**
